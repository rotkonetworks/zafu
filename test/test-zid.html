ZID — identity signing example

ZID lets websites prove who a user is without passwords.
The wallet signs a challenge → website verifies the signature → user is authenticated.
Each site gets a unique key. Sites cannot link your identities across origins.

1
detect zafu wallet
Check if zafu is installed. Send a ping message — if it responds, the wallet is available. 
// detect zafu
chrome.runtime.sendMessage(EXTENSION_ID, {
  type: "ping"
}, (resp) => {
  if (resp?.zafu) {
    // zafu is installed, version = resp.version
  }
});
2
authenticate user
Generate a random challenge and ask zafu to sign it.
The wallet shows an approval popup — user reviews and confirms.
On success, you get a signature and the user's public key for your site. 
// generate random 32-byte challenge
const challenge = new Uint8Array(32);
crypto.getRandomValues(challenge);
const challengeHex = bytesToHex(challenge);

// request identity signature
chrome.runtime.sendMessage(EXTENSION_ID, {
  type: "zafu_sign",
  challengeHex: challengeHex,
  statement: "Log in to My App",  // shown to user
}, (resp) => {
  if (resp?.success) {
    // resp.publicKey = user's ed25519 pubkey (hex, unique per site)
    // resp.signature = ed25519 signature of your challenge (hex)
  }
});
3
verify signature (server-side)
Verify the ed25519 signature against the challenge you sent.
In production, do this on your server — never trust the client.
The public key is the user's persistent identity on your site. 
// verify ed25519 signature (server-side in production)
const key = await crypto.subtle.importKey(
  "raw", pubkeyBytes, "Ed25519", false, ["verify"]
);
const valid = await crypto.subtle.verify(
  "Ed25519", key, signatureBytes, challengeBytes
);
// valid === true → user is authenticated
// store publicKey as the user's identity for your site
✓
authenticated identity
This is the user's persistent identity on your site. Same user = same pubkey every time.
Different site = different pubkey (unlinkable).
